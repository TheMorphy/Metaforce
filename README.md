# Metaforce

## Обзор проекта

Время на выполнение: 6-8 часов
Версия Unity 6000.3.5f1

Этот проект представляет собой небольшой игровой прототип, реализованный в рамках тестового задания на позицию **Senior Unity Developer**.

Проект реализован как тестовое задание, поэтому архитектура остаётся достаточно простой и не содержит избыточных абстракций, но несмотря на простоту реализации (формат тестового задания), архитектура проекта остаётся достаточно гибкой и расширяемой

В прототипе реализована следующая игровая логика:

* **Игрок управляется с помощью WASD**.
* Когда игрок **останавливается**, персонаж автоматически **находит ближайшего врага в радиусе атаки и стреляет по нему**.
* Враги **патрулируют между заранее заданными точками**.
* У врагов есть **HP и индикатор здоровья над головой**.
* После смерти враг **возрождается через заданное время**.
* В UI отображается **реактивный счётчик убитых врагов**.

Целью проекта было не только реализовать игровую механику, но и продемонстрировать **чистую архитектуру, модульность и расширяемость системы**.

# Используемые технологии

Проект использует следующие инструменты и библиотеки:

* **Unity**
* **Zenject** — dependency injection
* **R3** — реактивное программирование
* **DOTween** — анимации UI
* **ScriptableObject** — конфигурации

---

# Основные игровые системы

## Движение игрока

Отвечает за управление игроком.

Функции:

* чтение ввода через **новую Input System**
* перемещение персонажа через `CharacterController`
* публикация реактивного состояния `IsMoving`

Это состояние используется другими системами (например, системой автоатаки).

---

## Автоатака игрока

Система автоматически атакует врагов, когда игрок перестаёт двигаться.

Функции:

* поиск ближайшего врага через `Physics.OverlapSphere` (рассматривал NonAlloc, но решил в рамках тестового не делать)
* поворот игрока в сторону цели
* выстрел через **Raycast**
* запуск анимации стрельбы
* отображение визуального **tracer эффекта** с помощью `LineRenderer`

Цикл атаки реализован через **R3 Timer**, без использования Update-пулинга.

---

## Патрулирование врагов

Каждый враг перемещается между точками маршрута.

Функции:

* движение между патрульными точками
* плавный поворот по направлению движения

Маршрут передаётся врагу через **EnemySpawner**.

---

## Система здоровья врагов

Отвечает за получение урона и смерть врага.

Функции:

* хранение HP из `EnemyTypeConfig`
* применение урона
* отправка события смерти
* уничтожение объекта врага

Также отображает **полосу здоровья над головой**.

---

## Спавнер врагов

Отвечает за создание врагов и их возрождение.

Функции:

* начальный спавн врагов до `MaxEnemiesOnLevel`
* отслеживание смерти врагов
* запуск **индивидуального таймера респауна** для каждого убитого врага

Каждый враг возрождается **через своё время**, а не группой.

---

## UI — счётчик убийств

UI элемент, который показывает количество убитых врагов.

Поток событий:

```
EnemyHealth
      ↓
GameEvents.EnemyKilled
      ↓
KillCounter (реактивная модель)
      ↓
KillCounterView
```

UI автоматически обновляется при изменении значения.

# Система конфигураций

Конфигурация игры реализована через **ScriptableObject**.

Используются два типа конфигов:

### EnemyTypeConfig

Описывает тип врага:

* Prefab
* HP
* Скорость движения
* Скорость поворота

---

### EnemySpawnConfig

Описывает правила спавна:

* тип врага
* максимальное количество врагов на уровне
* время респауна

Это позволяет:

* использовать один тип врага на разных уровнях
* изменять правила спавна без изменения данных врага

---

# Реактивное программирование

Библиотека **R3** используется для обработки игровых событий и таймеров.

Реактивный подход используется для:

* состояния движения игрока
* счётчика убийств
* таймеров респауна
* передачи игровых событий

Преимущества:

* меньше логики в Update
* понятный поток событий
* слабая связанность систем

---

# Dependency Injection

**Zenject** используется для внедрения зависимостей.

DI применяется для:

* глобальных сервисов (`GameEvents`)
* конфигураций
* реактивных моделей

Это позволяет избежать скрытых зависимостей и улучшает модульность системы.

Локальные зависимости внутри одного объекта (например UI → EnemyHealth) получают через `GetComponent`.

# Визуальные элементы

## Полоса здоровья врага

Используется **World Space Canvas** с `Slider`.

При получении урона полоса здоровья **плавно уменьшается через DOTween**.

---

## Радиус атаки игрока

Радиус атаки отображается **визуальным кругом на земле**.

Это помогает игроку понимать дальность атаки.

# Итог

Проект демонстрирует:

* модульную архитектуру Unity
* использование dependency injection
* реактивный подход к игровым событиям
* конфигурацию через ScriptableObject

Основной фокус был на **читаемости, поддерживаемости и расширяемости кода**, при сохранении простой реализации, соответствующей формату тестового задания.
